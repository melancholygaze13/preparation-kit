# Navigation and Presentation

## Learning Path

### Rapid Review

1. [NavigationStack and Paths](navigation-stack-and-paths/README.md)
2. [Sheets, Popovers, and Dialogs](sheets-popovers-and-dialogs/README.md)

### Standard Preparation

Complete rapid review, then study:

3. [Destinations, Deep Links, and Restoration](destinations-deep-links-and-restoration/README.md)
4. [NavigationSplitView and Multicolumn Flows](navigation-split-view-and-multicolumn-flows/README.md)

All four concepts are core. For an imminent phone interview, prioritize the
single-column and modal state models. Multicolumn restoration and routing become
more important for Staff roles and iPad or macOS products.

## Concepts

| Concept | Why it matters | Priority | Time |
|---|---|---|---:|
| [NavigationStack and Paths](navigation-stack-and-paths/README.md) | Models pushed destinations as explicit state. | Core | 16 min |
| [Sheets, Popovers, and Dialogs](sheets-popovers-and-dialogs/README.md) | Defines ownership and routing for modal UI. | Core | 14 min |
| [Destinations, Deep Links, and Restoration](destinations-deep-links-and-restoration/README.md) | Connects external routes to validated, restorable navigation state. | Core | 16 min |
| [NavigationSplitView and Multicolumn Flows](navigation-split-view-and-multicolumn-flows/README.md) | Adapts selection and navigation across compact and expanded layouts. | Core | 15 min |
