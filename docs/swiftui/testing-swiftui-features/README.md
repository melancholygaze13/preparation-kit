# Testing SwiftUI Features

## Preparation Paths

- **Rapid review:** Study deterministic state tests, one critical UI flow, and
  accessibility audits.
- **Standard preparation:** Complete all four concepts in learning-path order.
- **Role-specific depth:** Add snapshot governance for design-system or
  component-platform roles, and lifecycle stress tests for async-heavy apps.

## Learning Path

1. [Testing State and Presentation Logic](testing-state-and-presentation-logic/README.md)
2. [Testing Navigation and Async Lifecycles](testing-navigation-and-async-lifecycles/README.md)
3. [UI and Accessibility Testing](ui-and-accessibility-testing/README.md)
4. [Snapshot Testing and Visual Regressions](snapshot-testing-and-visual-regressions/README.md)

## Concepts

| Concept | Why it matters | Priority | Time |
|---|---|---|---:|
| [Testing State and Presentation Logic](testing-state-and-presentation-logic/README.md) | Verifies deterministic feature decisions below the view layer. | High | 9 min |
| [Testing Navigation and Async Lifecycles](testing-navigation-and-async-lifecycles/README.md) | Exercises routing, cancellation, and timing boundaries. | High | 10 min |
| [UI and Accessibility Testing](ui-and-accessibility-testing/README.md) | Validates user-visible flows and semantic access. | High | 9 min |
| [Snapshot Testing and Visual Regressions](snapshot-testing-and-visual-regressions/README.md) | Detects selected visual changes with controlled baselines. | High | 8 min |
