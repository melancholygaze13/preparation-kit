# Language Basics

## Scope

The foundational rules Swift uses to represent values, express absence, enforce
types, report failures, and validate program assumptions. The organization
follows the Swift book's “The Basics” chapter, while each concept emphasizes the
decisions and failure modes relevant to experienced iOS engineers.

## Concepts

| Concept | Summary | Level |
|---|---|---|
| [Constants and Variables](constants-and-variables/README.md) | Bind names to values while making initialization and mutation explicit. | Senior |
| [Type Safety and Type Inference](type-safety-and-inference/README.md) | Understand how Swift establishes static types and uses context to solve expressions. | Senior |
| [Numeric Types and Conversions](numeric-types-and-conversions/README.md) | Choose numeric representations and make range, precision, and conversion behavior explicit. | Senior |
| [Tuples](tuples/README.md) | Group short-lived related values without introducing a nominal domain type. | Senior |
| [Optionals](optionals/README.md) | Model absence explicitly and choose an unwrapping strategy that preserves invariants. | Senior |
| [Memory Safety](memory-safety/README.md) | Preserve initialization, bounds, lifetime, and exclusive-access guarantees across safe and unsafe code. | Senior |
| [Error Handling](error-handling/README.md) | Represent recoverable failures, preserve context, and define recovery at the correct boundary. | Senior |
| [Assertions and Preconditions](assertions-and-preconditions/README.md) | Distinguish debug assumptions, production contracts, fatal invariants, and recoverable input failures. | Senior |
