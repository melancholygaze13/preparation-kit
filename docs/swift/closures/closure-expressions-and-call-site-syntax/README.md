# Closure Expressions and Call-Site Syntax

> Closure syntax is context-driven: Swift can infer types, returns, and shorthand
> arguments, while trailing closures reshape the call site. Use the shortest form
> that preserves roles, effects, and the boundary between arguments and control
> flow.
